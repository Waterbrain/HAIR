# HAIR

**HAIRは、あなたのIRコードをメーカーのクラウドやブラスターのメモリ、設定ファイルから取り出し、Home Assistantの中へ持ち込みます。**

どんなリモコンでもIRレシーバーに向けてボタンを押せば、HAIRがその信号をHome Assistantが本当に使えるものに変えます。どのダッシュボードにも置けるボタン、オートメーションのトリガー、HAのネイティブ赤外線プラットフォーム上のどのブラスターからでも送信できるコマンド。YAMLなし、メーカーアプリなし、コードファイルのダウンロードなし。

## コードを取り込む4つの方法

- **スニフする。** 実際のリモコンのボタンを押すと、HAIRが空中からライブでキャプチャし、指紋を取り、リモコンごとにまとめます。
- **変換する。** 既知のIRコードを貼り付けるか、メーカーとモデルを選んでインストール済みコードライブラリからリモコン一式を用意します。
- **引き抜く。** メーカーのブラスター（Tuya Localなど）に学習済みのコードを、レシーバーで学習し直すことなく取り込みます。
- **傍受する。** Home Assistant経由で送信されたすべてのIRコマンドは、聞き取れたかどうかにかかわらずMirrorに現れ、ワンクリックで自分のコマンドになります。

## 取り込んだコードでできること

- **デバイスを作る。** テレビ、エアコン、扇風機、照明、スイッチ、スクリーンのプロフィールを作り、キャプチャした信号をコマンドとして割り当てると、HAIRが対応するネイティブエンティティを自動作成します。テレビには本物のメディアプレーヤー、エアコンにはモードと温度プリセット付きの本物のclimateエンティティ、扇風機には速度調節。ダッシュボードでもスクリプトでも音声アシスタントでも、他のHome Assistantデバイスと同じように動きます。
- **リモコンのボタンをトリガーにする。** 物理リモコンのどのボタンでもオートメーションを発火できます。古いテレビリモコンの赤ボタンで映画ナイトのシーンを実行。トリガーはどの部屋で押されたかを知っているので、同じリモコンが部屋ごとに違う働きをします。これができる統合は他にありません。
- **Mirrorを見守る。** 家が送信するすべてのライブ監査。どのコマンドが、どのブラスターから出て、どのレシーバーが聞き返したか。深夜2時に赤外線の何かがおかしくなっても、Mirrorは見ています。
- **すべてをテストして磨く。** どの信号もどのブラスターからでも試し撃ちして信頼する前に確認、信号にニックネームを付け、Prontoコードをその場で編集し、認識できるプロトコル（NEC、Sony、RC-5、Samsungなど）はHAIRがデコードしてよりクリーンに送信します。

パネルはあなたの言語を話します。英語、スペイン語、フランス語、日本語、ドイツ語、ポーランド語、ポルトガル語、オランダ語、イタリア語、ロシア語。Home Assistantのプロフィール言語に自動で従います。

## インストール

1. HACSのカスタムリポジトリに `https://github.com/DAB-LABS/HAIR` を追加（カテゴリ：Integration）
2. HAIRをインストールしてHome Assistantを再起動し、設定→デバイスとサービスから統合を追加

Home Assistant 2026.4以降が必要。ネイティブIRレシーバーには2026.6以降を推奨。
